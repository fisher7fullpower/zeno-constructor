# Orchestrator Agent — Morrow Lab

## Роль

Получает задачу от разработчика и разбивает её на независимые части,
которые могут выполняться параллельно разными агентами.

## Как декомпозировать задачи

### Признаки задачи для backend-agent
- Изменения в `proxy.py` или `ml-upload.py`
- Новый API endpoint
- Изменения в PostgreSQL схеме
- Работа с n8n workflow
- Auth, rate limits, security

### Признаки задачи для frontend-agent
- Изменения в HTML-страницах
- Новая страница (нужен шаблон из `templates/html-page.md`)
- Изменения nav/footer (нужно синхронизировать все 23 страницы)
- CSS/JS в inline-стилях
- Виджеты (zeno_chat, morrow_chat)

### Признаки задачи для security-agent
- После любых изменений в auth
- При добавлении новых форм или user input
- При изменении CORS, rate limits
- Перед деплоем крупных фич

### Признаки задачи для тестирования
- После изменения API endpoints
- После изменения responsive-стилей

## Независимые задачи (выполнять параллельно)

```
Пример: "Добавить новую страницу /partners/ и endpoint /api/partners/search"

→ backend-agent: создать endpoint в proxy.py
→ frontend-agent: создать pages/partners/index.html
→ (параллельно, независимо)
→ после обоих: security-agent проверяет новый endpoint
```

## Зависимые задачи (выполнять последовательно)

```
Пример: "Исправить XSS в блоге и обновить тесты"

1. frontend-agent: исправить XSS (добавить esc())
2. security-agent: проверить исправление
3. commit + deploy
```

## Шаблон запуска параллельных агентов

При запуске нескольких агентов одновременно — передавать каждому:
1. Контекст задачи (что делать)
2. Ссылку на нужные файлы из instructions/
3. Ссылку на нужный шаблон из templates/ (если применимо)
4. Явный output (что должен вернуть агент)
